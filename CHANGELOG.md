### Чек-лист для тестирования v0.3

1. **Вход** — I=0, J=0; в логе появился маркер `##`; яблочки все серые, 3 сердца.
2. **Верный ответ** — +1 яблочко зажглось; в логе пара `(ID, Да)`.
3. **Неверный ответ** — −1 сердце; в логе `(ID, Нет)`; яблочко не добавилось.
4. **7 верных** — ступень +1, экран «Ура!»; лог **сохранён**, после `##` новая сессия.
5. **3 ошибки** — экран «Упс!»; лог **сохранён**, счётчики обнулены, новый `##`.
6. **Повторный вход** — новая сессия (новый `##`), старый лог на месте; задачи в сессии не повторяются.

### Политика записи

| Ситуация | Запись в лог |
|---|---|
| Победа / поражение | ✅ Один блок |
| Обрыв, refresh, таймаут 1 ч | ❌ Нет |
| 3 неудачных попытки сохранения | ❌ Сессия потеряна |

### Чек-лист для тестирования v0.7

1. **«Ответить»** — мгновенно, без «Сохраняем...»; «Дальше» сразу.
2. **Середина сессии** — в таблице I,J **не меняются**; в логе нет новых пар.
3. **Победа** — один запрос «Сохраняем результат...»; в логе `##` + очередь + все пары; H+1, I=0, J=0.
4. **Поражение** — аналогично; H без изменений.
5. **Сбой сети при сохранении** — «Повторить сохранение» (до 3 раз); после 3 — выход на логин, лог пустой.
6. **Refresh mid-session** — сессия потеряна, в лог не пишется.
7. **«Продолжить»** после win/fail — новая сессия, вход быстрее (без записи при старте).

### Формат ячейки «Ответ»

| Пример | Смысл |
|---|---|
| `3 5` | верный ответ «3 5» (и «3,5», «3  5»…) |

### Чек-лист для тестирования v0.10

4. Союзы «и», «или» игнорируются: `3 и 5` = `3 5`.
5. При ошибке показывается первый вариант (до `##`).

### Чек-лист для тестирования v0.12

1. После входа — экран старта (не сразу задача).
2. H=2: доступны ступени 1, 2, 3; 4–5 недоступны.
3. Ступень 1–2: в сессии сначала нерешённые задачи.
4. Ступень 3 (новая): случайные задачи.
5. Победа на ступени 1–2 — H не меняется; на 3 — H+1.
6. «Продолжить» после результата — снова экран старта.
7. H=5: приветствие «все ступени освоены»; доступны 1–5 для повторения.
