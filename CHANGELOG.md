## v0.2 (2026-07-04) — исправление выявленных багов

| Файл | Функция | Что изменено | Статус |
|---|---|---|---|
| `Code.gs` | `_getTasksByLevel` | Фильтр задач по **первой цифре** ID (ступень), вместо `indexOf`, который давал ложные совпадения | на тесте |
| `Index.html` | *(разметка)* | Убраны тестовые `value` у полей входа; placeholder «ID» вместо «ID или Имя» | на тесте |
| `Index.html` | `handleScriptFailure` | **Новая.** Скрывает лоадер и показывает alert при сбое `google.script.run` | на тесте |
| `Index.html` | `handleLogin` | Подключён `.withFailureHandler(handleScriptFailure)` | на тесте |
| `Index.html` | `loadNextTask` | Отображение «Ступень: N» = `studentLevel + 1` (решаемая ступень); `.withFailureHandler` | на тесте |
| `Index.html` | `handleSendAnswer` | Подключён `.withFailureHandler(handleScriptFailure)` | на тесте |
| `Index.html` | *(восстановление)* | Возвращены блок `quiz-stats` и функция `handleAnswerResponse` из полного исходника заказчика | на тесте |
| `Index.html` | `updateQuizStats` | **Новая.** Обновление счётчиков `quiz-stats` (без дублирования логики `handleAnswerResponse`) | на тесте |

### Чек-лист для тестирования v0.2

1. **Вход** — поля пустые, placeholder «ID»; вход только по ID из колонки A.
2. **Ступень на экране** — ученик с H=0 видит «Ступень: 1»; с H=2 — «Ступень: 3».
3. **Задачи ступени** — на листе «Задачи» задачи `101`, `102`… попадают только в ступень 1; `201`… — в ступень 2.
4. **Сбой сервера** — при отключённом интернете / ошибке GAS лоадер исчезает, показывается alert (не зависает «Загрузка...»).
5. **Основной цикл** — ответ → «Дальше» → новая задача; 7 задач / 3 ошибки — как в v0.1.
