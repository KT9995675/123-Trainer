**Формат ответа (v0.10):**
- Варианты через `##`: `3 5##5 3`
- Игнорируются: пробелы, регистр, пунктуация, союзы **и** / **или**

## 4. Архитектура (v0.10)

### Игра
- 5 ступеней, старт H=0, решает ступень H+1
- Победа: **7 верных**; провал: **3 ошибки**
- 10 задач в сессии (клиент), без повторов
- **Сессия на клиенте**; лог пишется **один раз** при win/fail (`finalizeSession`)
- Незавершённая сессия (refresh, таймаут 1 ч) — **в лог не пишется**
- До **3 попыток** сохранения при сбое сети

### UI (v0.8)
- Горизонтальный макет: **слева** управление, **справа** картинка/подсказка/ответ
- Шапка: **Тренажер "ВСЕМ ПЯТЬ!"** + логотип (base64 в CSS)

### Auth (v0.9)
- **Вход:** ID / телефон / почта + пароль
- **Регистрация:** имя, телефон, почта, пароль; класс/город/шола опц.
- **Восстановление:** 4 цифры на **почту** → вход → **смена пароля** (от 6 симв.)
- Лимит reset: **2 раза / 3 часа** (PropertiesService: `reset_{userId}`)
- Временный пароль: флаг `pwdTemp_{userId}` (без срока)
- Сброс лимита отладки: удалить ключи в **Настройки проекта → Свойства скрипта**

### Проверка ответов (v0.10, клиент)
- `normalizeAnswer`, `parseAnswerVariants`, `checkAnswerLocally` в **Index.html**
- Ответы в `sessionTasks[]` — **на клиенте** (по ТЗ)
