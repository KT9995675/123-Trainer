## 3. DEV и PROD (два контура)

| | DEV (отладка) | PROD (ученики) |
|---|---|---|
| Таблица | Копия боевой → `… — DEV` | Боевая таблица |
| Apps Script | Отдельный проект (из копии) | Боевой проект |
| URL веб-приложения | DEV URL (закладка разработчика) | URL для учеников |
| Данные | Тестовые пользователи | Реальные ученики |
| Развертывание | Можно «Последняя версия» | **Фиксированная версия** |

### Первичная настройка (один раз)

1. **Файл → Создать копию** боевой Google Таблицы → имя с суффиксом `DEV`.
2. В копии: **Расширения → Apps Script** — это DEV-проект (другой `scriptId`).
3. Развернуть DEV как веб-приложение (доступ: «Только я» или по необходимости).
4. В PROD: **Развернуть → Управление развертываниями** — боевое развертывание на **конкретной версии**, не «Последняя».
5. Заполнить `scripts/deploy.env` (см. §12).

### Цикл релиза

```
git (правки) → ./scripts/push-dev.sh → Новая версия в DEV → тест
            → ./scripts/push-prod.sh → Новая версия в PROD → сменить версию в развертывании
            → git commit + push (по команде)
```

## 5. Архитектура (v0.12)

### Игра
- 5 ступеней, H=0…5 (взятые ступени 1…H)
- После входа — **экран старта**: приветствие, статистика тем, выбор ступени
- Доступны ступени **1…(H+1)**; при H=5 — все 1–5 для повторения
- **Новая** ступень (H+1): 10 случайных задач
- **Взятая** ступень: сначала нерешённые учеником, потом остальные
- Победа: **7 верных**; провал: **3 ошибки**
- H растёт **только** при победе на ступени H+1; повторение — `practice_win`, H не меняется
- Сессия на клиенте; лог **один раз** при win/fail/practice_win (`finalizeSession`)
- Незавершённая сессия (refresh, таймаут 1 ч) — в лог не пишется

### UI
- Горизонтальный макет тренажёра; экран входа `card-auth`; экран старта `screen-home`
- Шапка: **Тренажер "ВСЕМ ПЯТЬ!"** + логотип (base64 в CSS)

### Auth (v0.9)
- Вход: ID / телефон / почта + пароль
- Регистрация, восстановление пароля на почту, смена временного пароля
- Лимит reset: 2 / 3 ч (`PropertiesService`)

### Проверка ответов (клиент)
- `normalizeAnswer`, `parseAnswerVariants`, `checkAnswerLocally` в **Index.html**

## 12. Деплой через clasp (рекомендуется)

### Установка (один раз)

```bash
npm install -g @google/clasp
clasp login
```

### Конфигурация

```bash
cd /Users/konstantin/Projects/math-trainer
cp scripts/deploy.env.example scripts/deploy.env
```

Заполните `scripts/deploy.env`:

| Переменная | Где взять |
|---|---|
| `DEV_SCRIPT_ID` | URL DEV Apps Script: `.../projects/SCRIPT_ID/edit` |
| `PROD_SCRIPT_ID` | URL PROD Apps Script |
| `DEV_WEBAPP_URL` | Развернуть → Управление развертываниями (для справки) |
| `PROD_WEBAPP_URL` | Боевой URL для учеников |

`scripts/deploy.env` в `.gitignore` — не коммитится.

### Пуш кода

```bash
chmod +x scripts/push-dev.sh scripts/push-prod.sh   # один раз

./scripts/push-dev.sh    # отладка
./scripts/push-prod.sh   # релиз (запросит подтверждение yes)
```

Скрипты отправляют в GAS только: `Code.gs`, `Index.html`, `appsscript.json` (см. `.claspignore`).

**Важно:** `clasp push` обновляет **код** в редакторе. Ученики видят изменения только после **Новая версия** + (для PROD) смены версии в боевом развертывании.

### PROD: закрепить версию

1. **Развернуть → Управление развертываниями**
2. Боевое развертывание → **Изменить**
3. Версия: выбрать номер (например `@15`), **не** «Последняя»
4. После каждого релиза: **Новая версия** → в развертывании указать новый номер

### Если clasp не используется

Копируйте три файла вручную в редактор DEV/PROD — workflow из §3 тот же.
